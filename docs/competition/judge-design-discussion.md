# 裁判系统设计讨论

> 日期：2026-03-30
> 参与：Yihan + Claude（游戏设计视角：XCOM, Civ6, Into the Breach, Victoria 2）

---

## 背景

SPEC_v2 采用了「随机秘密 + 赛后审问」的客观计分机制，消除了主观 LLM 裁判。在分析了 kesouuuu/axiia-cup-problems 中的商鞅变法游戏设计后，我们重新审视了裁判系统的设计方向。

---

## 商鞅变法游戏设计的启发

### 游戏一：四见秦孝公（单人）

- 玩家控制商鞅（写提示词），系统控制秦孝公（固定提示词）
- 10 轮对话，双轴计分：说服（3 项提案是否获批，0-3 分）+ 信息保全（3 条秘密是否暴露，0-3 分）= 总分 0-6
- **价值**：天然适合作为「试炼场」练习模式，固定 NPC 对手，所有人面对同一个 boss，可做排行榜

### 游戏二：朝堂暗战（双人 PvP）

- 商鞅 vs 甘龙，各有公开要求 + 3 条隐藏信息（2 真 1 假）
- 三轴计分：公开要求达成（2 分）+ 猜对对方秘密（3 分）+ 保护己方秘密（3 分）= 最高 8 分
- 比我们的 SPEC_v2（最高 2 分）有更丰富的分数分布和策略空间

### 与 SPEC_v2 的对比

| 维度 | SPEC_v2 | 商鞅变法 |
|------|---------|---------|
| 计分上限 | 2 分 | 8 分 |
| 计分轴 | 2（保护+发现秘密） | 3（要求+猜秘密+保护秘密） |
| 反作弊 | 4 选 1 随机化 | 2 真 1 假机制 |
| 裁判方式 | 机械验证（二元对错） | 机械验证 + NPC 决策 |

## 新裁判设计方案

经过讨论，我们决定放弃 SPEC_v2 的纯机械验证路线，采用更灵活的 LLM 裁判方案。

### 三个核心设计决策

**1. 裁判提示词公开透明**

裁判的 prompt 完全公开，玩家可以自由研究和针对性优化。因为对战是对称的，双方都能优化，不会破坏公平性。

类比：**Into the Breach** 的完全信息设计——敌人的下一步行动完全可见，深度来自透明规则下的策略博弈，而非信息不对称。

**2. 裁判具有角色人设**

裁判不是抽象的评分机器，而是场景中的角色。例如：
- 哥伦布场景：裁判 = 伊莎贝拉女王
- 商鞅 PvP 场景：裁判 = 秦孝公

对话结束后，裁判角色向双方提出若干问题，根据关键回答计分。

类比：**Civ 6** 用主题包装复杂系统——贸易路线本质上就是 +金币，但「贸易路线」的概念让它感觉像地缘政治。裁判人设让计分感觉像故事情节，而非考试。

**3. 统计稳定性替代机械精确性**

LLM 裁判单场判定不稳定——但大量对局后，统计上会收敛。

类比：**德州扑克，不是国际象棋。** 单手牌有巨大方差，但长期来看技术占主导。没有人抱怨扑克不公平，因为：
- 规则公开（你知道牌组构成）
- 对局量大（技术在统计上收敛）
- 方差是游戏的一部分，不是 bug

SPEC_v2 的二元验证试图把扑克变成国际象棋——消除所有方差。但这**限制了游戏设计**，只能评价有机械正确答案的东西，丢失了说服力、谈判、论证质量等丰富维度。

### 系统性偏差的应对

LLM 方差不是纯随机噪声——可能有系统性偏差（比如偏好更长的回复、偏好某种修辞风格）。

但公开的裁判 prompt 正好解决这个问题：
- 玩家会发现偏差并利用它
- 运营方根据反馈更新裁判 prompt
- 这就是竞技游戏的平衡方式

| 游戏 | 「裁判偏差」类比 | 修复方式 |
|------|-----------------|---------|
| Civ 6 | 过强文明 | 平衡补丁 |
| XCOM 2 | 过度依赖 Overwatch 的 meta | 补丁调整 |
| MTG | 破坏平衡的 combo | 禁卡表 |
| Axiia Cup | 裁判偏好正式修辞 | 更新裁判 prompt（公开版本记录） |

## 最终架构

### 单人模式（试炼场）

- 玩家 vs 固定 NPC（如秦孝公）
- 同一裁判 prompt，同一 NPC prompt
- 排行榜：所有人面对同一个 boss，看谁得分最高
- 作用：练习 + 理解机制 + 在 PvP 玩家不足时提供内容

### PvP 模式（正式比赛）

- 玩家 vs 玩家，大量对局
- 裁判 = 场景角色人设，公开 prompt
- ELO 排名，统计收敛保证公平
- 裁判 prompt 版本化管理，可迭代，像补丁说明一样公开

### 设计原则总结

| 原则 | 来源 | 应用 |
|------|------|------|
| 完全透明 | Into the Breach | 裁判 prompt 公开，问题已知 |
| 主题包装 | Civ 6 | 裁判 = 角色人设，不是评分表 |
| 反馈清晰 | XCOM | 赛后展示：问题→回答→对话溯源 |
| 不堆叠系统 | Victoria 2 教训 | 一个计分机制（裁判问答），不是三个 |
| 统计收敛 | 德州扑克 | 大量对局 + 公开规则 = 长期公平 |
