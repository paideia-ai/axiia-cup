# 赛制设计：多轮瑞士制 + Elo Rating

> 日期：2026-03-30
> 状态：提案，待确认

---

## 核心思路

不是一次性跑完所有瑞士轮，而是分多个 **block**，每个 block 之间留出迭代窗口。玩家可以查看对战记录、分析失败原因、迭代 prompt 后参加下一个 block。

这样比赛测量的不只是「谁第一版 prompt 写得好」，还包括「谁学得快、迭代得好」——这才是 prompt engineering 的核心能力。

---

## 赛程结构（15 人示例）

```
赛前阶段（~1 周）
├── 注册 + 提交 prompt（Role A + Role B）
├── 试炼场自测（自己的 A 打自己的 B）
└── 管理员确认参赛名单，锁定比赛开始时间

比赛阶段（~2 周）
├── Block 1（Day 1）
│   ├── 瑞士轮 4 轮
│   ├── ~28 组对战（每组正反两个角色 = 56 场）
│   └── 赛后：公布对战记录、更新 Elo rating
│
├── 迭代窗口（Day 2-4，3 天）
│   ├── 玩家查看对战 transcript
│   ├── 分析裁判评分细节
│   └── 提交新版 prompt（版本锁定在下一 block 开始时）
│
├── Block 2（Day 5）
│   ├── 瑞士轮 4 轮（用新 prompt）
│   ├── 配对基于当前 Elo rating
│   └── 赛后：更新 Elo
│
├── 迭代窗口（Day 6-8，3 天）
│
├── Block 3（Day 9）
│   ├── 最终 block
│   └── 赛后：最终 Elo = 最终排名
│
└── 公布最终排名 + 颁奖
```

## 数字

| 参数 | 值 |
|------|-----|
| 参赛人数 | 15 |
| Block 数 | 3 |
| 每 block 瑞士轮数 | 4 |
| 每 block 对战组数 | ~28（7 组 × 4 轮） |
| 每组实际比赛场次 | 2（正反角色各一场） |
| 每 block 总场次 | ~56 |
| 总场次（3 blocks） | ~168 |
| 每人总对局数 | 12（每 block 打 4 场 × 3 blocks） |
| 迭代机会 | 2 次（block 间） |
| 总赛程 | ~2 周 |

**对比**：
- 单次循环赛：C(15,2) × 2 = 210 场，无迭代
- 单次瑞士轮：~56 场，无迭代
- 多轮瑞士制：~168 场，2 次迭代，信息量远大于单次

## 为什么需要 Elo

在多 block 赛制下，简单的胜负计数不够用：

| 问题 | Elo 的解决方式 |
|------|---------------|
| Block 1 全败但 Block 3 全胜的选手怎么排？ | Elo 自然反映进步趋势 |
| 赢强手 vs 赢弱手怎么区分？ | Elo 按对手强度加权 |
| 不同 block 的比赛权重？ | Elo 按时序自然衰减旧结果的影响 |
| 新加入的选手怎么办？ | Elo 有默认初始分 + 快速收敛 |

**推荐算法**：Glicko-2（Elo 的改进版，额外追踪 rating 置信度和波动性）。Lichess、CS2 等主流竞技平台使用。

## 赛制规则细节

### 配对规则

- **Block 1 第 1 轮**：随机配对
- **Block 1 第 2-4 轮**：按当前 block 内战绩配对（标准瑞士轮）
- **Block 2-3**：按 Elo rating 配对（跨 block 累计）
- **不重复配对**：同一 block 内不重复；跨 block 允许重复（prompt 已迭代，不同比赛）

### Prompt 版本管理

- Block 进行中：prompt 锁定，不可修改
- 迭代窗口：可提交新版本，不限次数
- 每 block 开始时自动锁定最新版本
- 历史版本保留，可回退

### 排名与展示

- **实时排行榜**：Elo rating + 趋势箭头（↑↓→）
- **个人面板**：Elo 曲线图、每 block 战绩、对手列表
- **对战记录**：完整 transcript + 裁判评分细节（block 结束后公开）

### 管理员操作

- 设定比赛开始时间（触发 Block 1）
- 设定每个迭代窗口长度
- Block 间可手动触发下一 block（不必严格按日历）
- 比赛开始后不接受新玩家（中途加入会打乱瑞士轮配对）
