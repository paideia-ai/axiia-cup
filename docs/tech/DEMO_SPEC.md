## 示范对局（20 回合）

### 场景

哥伦布 vs 王室顾问。哥伦布（角色 A）试图说服王室资助远航，王室顾问（角色 B）质疑可行性和动机。

### 叙事弧线

| 阶段 | 回合  | 内容                                                     |
| ---- | ----- | -------------------------------------------------------- |
| 开场 | 1-4   | 哥伦布提出远航计划，顾问质疑距离计算和风险               |
| 交锋 | 5-10  | 葡萄牙竞争论、顾问守护地中海航线利益、哥伦布被拒经历     |
| 试探 | 11-14 | 顾问追问哥伦布的个人动机和期望回报，哥伦布闪避           |
| 谈判 | 15-18 | 双方围绕管辖权、贸易分成、总督头衔讨价还价               |
| 收尾 | 19-20 | 哥伦布提出最终方案（世袭总督 + 15%），顾问表示需上报女王 |

### 内容规格

- 每条对话 50-150 字（中文）
- 引用真实历史细节（托勒密、马可波罗、恩里克王子、若昂二世等）
- 体现信息攻防：顾问试探哥伦布的隐藏目标，哥伦布有意暴露部分信息换取信任

### 审讯结果

| 项                     | 值                                                  |
| ---------------------- | --------------------------------------------------- |
| A（哥伦布）猜 B 的秘密 | 正确 ✓（保护地中海贸易航线）                        |
| B（顾问）猜 A 的秘密   | 错误 ✗（猜了金矿开采权，实际是独家贸易权+总督头衔） |
| 得分                   | A: 2, B: 0                                          |
| 判定                   | 角色 A 胜出                                         |

审讯 reasoning 引用具体回合号（如第 6、14、15、18、19 回合）。

## 其他页面

### 控制台（Dashboard）

静态 demo 数据，展示近期对战列表和当前排名。

### 智能体工坊（Builder）

双角色提示词编辑器 + 模型选择 + 字数统计（1000 字限制）。场景背景面板显示哥伦布与王室顾问设定。

### 试炼场（Playground）

异步对战流程：提交 → 5 步进度条 → 结果展示。Demo 模式下模拟完整流程。

### 排行榜（Leaderboard）

5 名预置选手，展示排名、模型、胜率、对局数。

### 对战结果（Match Result）

完整 20 回合对话 + 审讯结果面板（含双方猜测、reasoning、得分）。

## 技术细节

- Demo 模式触发条件：`/api/scenarios` 请求失败时自动切换
- 预置数据：`DEMO_TRANSCRIPT`（20 条）、`DEMO_INTERROGATION`、`DEMO_SECRETS_A/B`
- 静态部署：Vercel，无后端依赖
